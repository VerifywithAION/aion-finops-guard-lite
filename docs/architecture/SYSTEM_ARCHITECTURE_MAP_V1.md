# System Architecture Map V1

## Purpose

This document defines the system role of AION FinOps Guard Lite.

---

## Core role

AION FinOps Guard Lite sits between financial intent and financial consequence.

It is a pre-execution decision layer.

---

## Canonical flow

1. Intent arrives
2. Preview is formed
3. Decision is produced
4. Receipt is emitted
5. Execution is allowed, warned, or blocked

---

## Current Lite surfaces

- minimal runtime wedge
- local open-source governance mode
- imported external billing snapshot path
- Buzz-compatible import path
- API-connectable Lite surface
