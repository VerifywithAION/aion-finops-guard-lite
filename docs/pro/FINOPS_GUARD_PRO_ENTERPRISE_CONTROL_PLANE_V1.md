# FinOps Guard Pro Enterprise Control Plane V1

## Purpose

This document defines the enterprise control plane for Pro.

---

## Required surfaces

- org / team / project hierarchy
- budget domains
- provider allowlists
- approval routing
- risk tiers
- policy simulation before rollout
- shadow mode
- warn-only mode
- block mode

---

## Why it matters

Real enterprise adoption requires rollout control, hierarchy, and staged enforcement.

This layer makes the system deployable at larger scale.
